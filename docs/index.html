Documentation D-02

Component Map

All UI components, their purpose, props, and interaction behaviour. Covers both the form and results sections.

🗺️ TripPlannerForm
Purpose: Main form container. Manages all state and submits to backend.

State:
  • formData (destination, duration, budget, groupSize, travelStyle, interests)
  • itinerary, loading, error
  • popularDestinations (from API)

API calls:
  • GET /api/popular-destinations (on mount)
  • POST /api/generate-itinerary (on submit)
🎯 Interest Pills
Purpose: Multi-select interest toggle. Each pill is an independent toggle button.

Behaviour:
  • Click to select (saffron + ✓)
  • Click again to deselect
  • Multiple can be active
  • Defaults to ['sightseeing','culture'] if none selected

Options: Heritage, Adventure, Nightlife, Nature, Culture, Food, Shopping, Relaxation, Photography, Spiritual
⚖️ Travel Style Selector
Purpose: Single-select for travel budget style.

Values sent to API:
  • 'budget' → Budget Traveler
  • 'balanced' → Balanced Experience (default)
  • 'luxury' → Luxury Travel

Layout: 3-column grid on desktop, vertical stack on mobile
📅 ItineraryResults (inline)
Purpose: Renders the full AI response. Embedded in TripPlannerForm, conditionally shown when itinerary state is set.

Props:
  • itinerary (full JSON from API)
  • preferences (form values for meta display)

Sections rendered: Hero banner, Budget breakdown, Daily itinerary, Accommodation, Highlights, Local tips, CTA
🏨 ItineraryDisplay
Purpose: Standalone component for displaying a pre-fetched itinerary outside the form flow.

Props:
  • itinerary (same JSON structure)

Difference from ItineraryResults: No preferences display, no hero banner. Just the day-by-day view.
⚙️ groqService.js
Purpose: Backend service. Builds prompt, calls Groq API, parses JSON, returns itinerary or fallback.

Model: llama-3.3-70b-versatile

Error handling:
  • 401/403 → Invalid API key message
  • 429 → Rate limit message
  • JSON parse fail → fallback itinerary
  • Empty response → fallback itinerary
